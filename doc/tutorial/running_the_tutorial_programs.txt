The tutorial programs don't do any parameter checking or error handling.
This is intentional, the goals is to have simple and understandable demo's

The source for the programs lives in tests/tutorial.
-----------------------------------------------------------------

program 1: RemoteCopy
---------------------

This program copies files. it is only 6 lines, but very powerful, as shown
with the examples below.

local to local
bin/run_gat_app tutorial.RemoteCopy /bin/echo foo

remote to local with sftp
bin/run_gat_app tutorial.RemoteCopy sftp://kits.cs.vu.nl//bin/echo foo

remote to local with gsiftp
bin/run_gat_app tutorial.RemoteCopy gsiftp://fs1.das3.liacs.nl//bin/echo foo


remote to local with any protocol (try -Dgat.verbose to find out which is used)
bin/run_gat_app tutorial.RemoteCopy \
  any://kits.cs.vu.nl//bin/echo \
  foo

error in the URI, to show GAT error reporting
bin/run_gat_app tutorial.RemoteCopy ssj://kits.cs.vu.nl//bin/echo foo

bin/run_gat_app tutorial.RemoteCopy \
  /bin/echo \
  any://kits.cs.vu.nl/foo


bin/run_gat_app tutorial.RemoteCopy \
  any://fs1.das3.liacs.nl//bin/echo \
  any://fs2.das3.science.uva.nl/hiha.dat

(Login to fs2 to show that the file actually ended up there.) 
Note that any://fs1.das3.liacs.nl//bin/echo is an absolute path, while
any://fs2.das3.science.uva.nl/hiha.dat is a relative path.
