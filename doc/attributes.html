Globus

attribute keypossible value(s)default valuemeaning
"globus.exitvalue.enable"“true”, “false”“false”globus doesn't offer the exit status, so by default JavaGAT assumes that the job had run correctly and will return an exit status of 0 for a globus job. However using a wrapper script that starts the real application and writes the exit value of that application in a file, we can retrieve the exit value. If you want to use this, set this attribute to true. Note that it has overhead: creation and prestaging of the wrapper script, post staging and reading of the file containing the exit value. The filename used for the wrapperscript is “.JavaGAT-wrapper-script-” and for the exit value file “.JavaGAT-exit-value-” where in the wrapper script file name is equal as in the exit value file name. Also note that for instance a submission of a job containing a non-existing executable without this preference, will cause a submission error, while submitting the same job with the preference set to true, won't result in a submission error, but in a proper submission, however the exit value then indicates that the executable didn't exist (exit value 127).
"globus.queue"queue name-the name of the queue where the job should be submitted to
