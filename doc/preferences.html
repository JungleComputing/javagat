Use of adaptor name list

The preference "XXX.adaptor.name" requires an adaptor name list string as value. Multiple adaptors can be provided where the first provided adaptor will be tried first, then the second etc. When a particular adaptor should be excluded, you have to provide a '!' before its name. An empty adaptor name means that the remaining adaptors follow in random order. Below are a few examples that explain the usage. They all operate on the example set of adaptors: "ssh", "ftp", "sftp", "local", "gridftp"
